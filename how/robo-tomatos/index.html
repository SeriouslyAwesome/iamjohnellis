How To: Boost Productivity with Robotic Tomatos

July 25, 2014

Productivy Help from Robotic Tomatos

…anyone can do any amount of work, provided it isn’t the work he is supposed to be doing at that moment.

– Robert Benchley, in Chips off the Old Benchley, 1949

In a bizarre bout of meta-productivity, or what some might call “structured procrastination,” I’ve been procrastinating on actual work to put together a workflow for more productive work periods. Although I’m a little late to the game, I’m giving the popular Pomodoro TechniqueTM a shot. But I’m very easily distracted by a certain website, and it’s getting in the way of my tomatos.

A few years ago, I found an aptly-named app called Concentrate that promised to help you get to work by automatically launching applications, quitting other applications, and blocking distracting websites all with a single click. Sadly, Concentrate seems to have been left behind by its developers, and my copy has suddenly stopped working altogether.

While there are no direct replacements, there is an app that can actually do everything Concentrate did, and tons more. Best of all, if you’re on a Mac, it’s free, and it’s already installed on your system.

Enter Automator

Automator is probably well known to the casual productivity nerd, but for the uninitiated, you can think of Automator as an intern for your Mac. Any task you might use your silly mouse or keyboard to do manually (say, open an application, or create an iPhoto album, or combine PDF pages, etc.), you can quickly drag-n-drop into a chain that Automator will run automatically.

What follows is an example of how I’m using Automator to replicate the functionality of the now defunkt Concentrate, while also integrating the Pomodoro TechniqueTM. The possibilities are damn-near infinite though, so use this as a starting point and tweak, expand, and hack to suit your needs.

The Setup

Your needs will undoubtedly vary, but it helps to start with an outline of what you typically do when you sit down to work. For example, when I’m doing Ruby/Rails development, I typically do the following:

  • Open Textmate
    • Open project folder
  • Open iTerm
    • cd into project directory in two terminals
  • Open GitHub for Mac
    • Select project repo
  • Open PG Commander
  • Open Pandora

If I’m on a deadline and really need to focus, I’ll also:

  • Quit Messages
  • Quit Mail

When I was using Concentrate, I could blacklist whatever websites I found to be counterproductive. Focus is a free, minimal, and actively-developed alternative. So let’s now add to our workflow:

  • Turn on Focus

So far, that’s already eleven steps! Just to get ready to work! The whole point of software is to automate repetitive tasks, so let’s employ Automator to do what it was born to do!

ROBOT, ACTIVATE!

[Image: Open Automator and create an Application]

You’ll want to start by opening Automator and creating a new Application. An Application can receive files or folders as input, and do something with them throughout the workflow. So by the end of this, if I drag a project folder onto my Automator app, I’d like the following to happen (automatically):

  • Quit all running applications (except for the ones we’re about to use, if they’re already open)
  • Take the folder we dragged in, and open it in:
    • Textmate
    • iTerm
    • GitHub
  • Open PG Commander
  • Open Pandora
  • Turn on Focus

Here’s what this looks like in Automator so far:

[Image: Illustration of Above Actions] Note: You’ll notice Focus is activated by a shell script: open focus://focus - at the time of this writing, this is only available with the latest alpha build on GitHub. In testing up this workflow, running a shell script seemed to stop the chain from processing the folder I dropped on the workflow, so I moved the script to the end.

Great! We’ve now reduced eleven steps down to one! If that’s not impressing you yet, let’s move on to integrating our Pomodoro TechniqueTM.

ROBO-TOMATO, ACTIVATE!

The standard workflow is 25 minutes of productivity, with a 5-minute reward of free time. So now that we’re up and running with our project folder, and our internet distractions are blocked, let’s tell Automator to pause itself for the next 25 minutes:

[Image: Pause for 25 minutes]

Like a good little programmer, I’m going to DRY up my workflow by putting my break time activites into a seperate workflow. Why? Because I can run one Automator workflow within another. I’m building a workflow for Ruby/Rails work right now, but I might want to duplicate this and modify it for writing time, or maybe PHP/Wordpress development, etc. By making a ‘Break Time’ workflow that these other workflows can run, I won’t have to recreate that part of the chain every time. Plus if I decide to tweak what I allow myself on break, I only have to edit it once, and the primary workflows will respond accordingly.

Here’s what my five-minute break looks like:

[Image: Break Time workflow]

This is introducing some interesting interactions:

  • When the break starts, make a sound (made possible by additional actions imported into my library from this website)
  • Display a notification alerting me that it’s now break time.
  • Turn off Focus to allow distracting websites
  • Launch Mail
  • Open Reddit! Hooray!
  • Pause for another five minutes
  • Then play a sound again when the break is over

Rolling this back into our primary workflow is easy enough, but now we’ll also add an action called ‘Ask for Confirmation’, which offers to continue the chain (if I click the affirmative button), or abort the chain (if I click the negative). In our case, the next step in the chain is to loop back around to the start, at which point we start a new Pomodoro:

[Image: Final Setup]

You might notice that you can configure your Loop action to only loop a certain number of times. If you’re really sticking to the Pomodoro TechniqueTM, you could use this to run through four pomodori before triggering your longer 15-30 minute break workflow. I’ll leave it to implement that anyway you wish.

So Productive Right Now

So with that, we’ve now got a very quick and easy way to not only get up and running with a project, but stay focused on it for 25 minutes at time, automatically give yourself a break, and repeat the cycle until you feel like stopping.

Automator includes a pretty impressive library of actions that can be performed by a number of different apps, and there are even more you can download. I advise you to do some Googling and see what else is possible. You’re already using the computer to do (mostly) interesting things, but with the combinations that are possible with Automator, your computer can do even more to help you than you probably realize. Make that computer your bitch. Or intern. Or tomato. Whatever tickles your fancy.
